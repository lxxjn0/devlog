## 학습 테스트(*Learning Test*)란

프로젝트를 진행하다 보면 새로운 API나 사용해본 적이 없는 라이브러리를 사용해야 하는 때가 찾아온다.

기존에 사용법이 친숙한 라이브러리라면 어떻게 사용하는지, 또 그에 대한 동작이 어떻게 이루어지는지 예측할 수 있다. 하지만 처음 사용해 보는 것이라면 이야기가 다르다.

그래서 우리는 프로젝트를 적용하기 이전에 새롭게 적용할 API나 라이브러리가 예상대로 동작하는지 확인해 볼 필요가 있다.

외부 라이브러리를 확인해보는 방법 중 하나가 바로 **학습 테스트(*Learning Test*)**이다.

학습 테스트는 기능 구현을 위한 테스트라기보다 새로운 API, 라이브러리, 프레임워크가 **어떻게 동작하는지 이해하기 위해** 작성하는 테스트 코드이다.

## 학습 테스트의 장점

처음 학습 테스트를 작성하는 경우 그리 쉽지만은 않을 것이다. 그러나 만약 테스트를 작성하는데 익숙하다면 학습 테스트는 여러 장점들을 우리에게 가져다준다.

학습 테스트의 장점으로는 아래와 같다.

- 다양한 조건에 따른 기능을 손쉽게 확인해 볼 수 있다.

- 학습 테스트 코드를 개발 중에 참고할 수 있다.

    - API 호출 방법과 호출에 따른 결과를 알 수 있기 때문이다.

- 프레임워크나 제품을 업그레이드할 때 호환성 검증을 보여준다.

    - 추후 프로젝트에서 API와 같은 것들의 버전을 업그레이드 하더라도 학습테스트가 작성되어 있다면 이를 통해 호환성을 보장할 수 있다.

- 테스트 작성에 대한 좋은 훈련이 된다.

- 새로운 기술을 공부하는 과정이 즐거워진다.

## 정리

학습 테스트는 새로운 API나 라이브러리를 재밌게 익힐 수 있는 아주 좋은 방법이다.

또한, 학습 테스트를 작성함으로서 테스트 코드가 더 익숙해지고 가까워질 수 있다.

**좋은 개발자가 되기 위해서 테스트는 땔래야 땔 수 없는 부분이라고 생각한다.** 그렇기 때문에 다양한 학습을 테스트를 통해 진행하는 것은 분명 큰 도움이 될 것이 틀림없다.
